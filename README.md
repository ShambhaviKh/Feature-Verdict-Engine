# Feature Verdict Engine

A single-purpose tool that takes a consumer app name and a proposed feature idea and returns a structured product-management audit — a verdict, five weighted scores, a competitive read, and a critique of your success metric — generated by an LLM (Grok, via xAI's API) called directly from the browser.

## How to run it locally

1. Download/clone the three files: `index.html`, `style.css`, `app.js`.
2. Get an xAI API key from [console.x.ai](https://console.x.ai).
3. Open `index.html` in a browser (or use VS Code's "Live Server" extension for a nicer dev loop).
4. Paste your API key into the field at the top of the page (it's kept in memory for the tab only — nothing is persisted or sent anywhere except `api.x.ai`).
5. Fill in the app name and feature idea, optionally a success metric, and click **Run audit**.

No build step, no `npm install`, no backend — it's three static files.

> **Note on CORS:** some APIs block direct browser calls unless you're on a paid/verified key tier. If you see a network/CORS error in the console, you'll need a tiny local proxy (e.g. a 10-line Express or Flask route that forwards to `https://api.x.ai/v1/chat/completions`) rather than calling it straight from the page. That's the only reason a backend would need to exist here.

## Why this project

This tool operationalizes PM judgment — feasibility, novelty, metric design, competitive positioning — into a repeatable, structured audit format, rather than just producing another paragraph of written analysis. It forces the same rigor (a verdict, a scored breakdown, a metric critique, a named next step) onto every idea it's pointed at, which is what makes it useful for quickly triaging a backlog of feature pitches rather than writing a fresh memo for each one.
